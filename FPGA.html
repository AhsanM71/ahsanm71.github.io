Project Details

Introduction

This project involves designing and implementing a combination lock on an FPGA, programmed using Verilog. The lock system works by comparing user input with a predefined combination and provides feedback on whether the correct combination was entered or not. The system includes features such as the ability to set a new combination and alarm functionality if the combination is incorrect.

Module Breakdown

1. Comparator Module

The comparator module is responsible for comparing the user input with the predefined combination. If the input matches the combination, the lock is "opened" and the system signals a success. If not, the user is informed that the input is incorrect.

2. 7-Segment Display Module (hexDisp)

This module controls the display of information using a 7-segment display. Based on the current state of the lock (e.g., open, new combination, or alarm triggered), it shows the appropriate output, providing feedback to the user.

3. Main Control Module

The main module ties together all the sub-modules such as the comparator, display, and register modules. It manages the overall operation of the lock, including reading user input, comparing it to the combination, and handling alarms and new combination settings.

4. Pulse Circuit Module

The pulse circuit ensures that button presses (e.g., Enter and Change) are processed properly, generating pulse signals for other components to act upon. It helps in filtering the noise of button presses and ensures that only valid presses trigger the appropriate actions.

5. Register and Combination Register Modules

These modules store the current combination and allow updates if the user wants to change the lock's combination. The combination is stored as a 4-bit value, and the system allows the lock to be reset and new combinations to be set by the user.

How the Combination Lock Works

  1. The user inputs a 4-digit combination via input switches, which is fed to the comparator module.
  2. The comparator checks whether the user input matches the stored combination.
  3. If the combination matches, the lock is opened, and a signal is sent to the display to indicate success.
  4. If the combination is incorrect, an alarm is triggered after a certain number of attempts.
  5. Using the "Change" button, the user can reset the combination and store a new one in the system.
  6. All outputs (success, alarm, or new combination) are displayed on the 7-segment display for clear feedback.

Key Features

  • Combination Matching: Compares the input combination with the stored combination and opens the lock if correct.
  • New Combination Setting: Allows users to set a new 4-bit combination using a dedicated button.
  • Alarm System: Triggers an alarm if the wrong combination is entered multiple times.
  • Visual Feedback: Provides real-time feedback through a 7-segment display, indicating the lock's status.
