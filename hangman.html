Project Details

Introduction

This project is a Hangman game developed using Object-Oriented Programming (OOP) principles in Java. The game implements several design patterns, such as SOLID, GRASP, and encapsulation, ensuring a well-structured and maintainable codebase. The graphical user interface (GUI) was designed for an interactive and user-friendly experience, providing different levels of difficulty for players.

Game Features

The Hangman game offers the following key features:

  • Difficulty Levels: Users can select between Easy Mode and Hard Mode from the homepage. Easy mode offers more lives and simpler words, while Hard mode provides fewer lives and more challenging words.
  • Help Section: A dedicated help option on the homepage provides instructions on how to play the game.
  • Word Guessing: Players are tasked with guessing a hidden word by inputting one letter at a time. If the guessed letter is correct, it appears in the word. If incorrect, the player loses a life and a limb of the hangman appears. The game ends when the player either guesses the entire word or runs out of lives and the entire hangman character is drawn.

How Hangman Works

In the game of Hangman, a hidden word is represented by blank spaces, and players attempt to guess letters to reveal the word. Each correct letter guessed is revealed in its respective position, while incorrect guesses reduce the number of remaining lives. The objective is to guess the full word before running out of lives.

Design Principles

This project was built using several key software design principles:

  • SOLID: The game adheres to SOLID principles, ensuring that each class has a single responsibility and that the code is easily extendable and maintainable.
  • GRASP: The General Responsibility Assignment Software Patterns (GRASP) were applied to assign clear responsibilities to each class, ensuring a modular and efficient design.
