Project Details

Introduction

Our team has developed a comprehensive solution for ACME Corp's Rescue Drone System, designed to manage the drone's operations for locating and rescuing individuals in hostile marine environments. The system consists of four main components: the Drone package, the Map package, the Phase package, and the Report package.

Drone Package Overview

The Drone package handles the conversion of drone commands from JSON format into executable Java code, utilizing the Adapter Pattern from the Gang of Four design patterns. Key aspects include:

1. JSON to Java Command Transformation

Transforms JSON commands into Java objects for execution, integrating external command sources with our Java-based system.

2. Adapter Pattern Implementation

Utilizes the Adapter Pattern to ensure compatibility between command sources and the internal control system, providing a consistent interface and enhancing flexibility.

Map Package Overview

The Map package is crucial for tracking the geographic information relevant to the rescue mission. It is responsible for:

1. Tracking Inlets and Emergency Sites

Maintains records of discovered inlets and emergency sites during the drone's scans of the island.

2. Finding the Closest Inlet

Determines the nearest inlet to the emergency site upon completion of scanning or battery depletion, ensuring efficient rescue operations.

Phase Package Overview

The Phase package contains the algorithm phases that guide the drone's scanning of the island. The phases are structured as follows:

1. Initial Position to Nearest Corner

Moves the drone to the nearest corner from its initial position to calculate distances of inlets and the emergency site relative to a corner of the map.

2. Initial Detection of Island

In this phase of the algorithm, the drone traverses and scans the map to identify the island's edge.

3. Scanning Phases

The following phases are executed in a loop until the entire map is scanned or the drone runs out of battery:

  • ScanLine Phase: Fly and scan the island from one side to the other.
  • UTurn Phase: Turns the drone around when it reaches the edge of the island, allowing for continuous scanning.

These phases repeat until the scanning objective is completed or the drone’s battery is depleted.

Report Package Overview

The Report package is responsible for generating reports at the end of the program. This includes:

1. Mission Success Check

Determines whether the drone successfully found the nearest inlet to the emergency site or if it failed to complete the mission.

2. Final Reporting

Outputs the result of the mission, providing information on the success or failure of the rescue operation and ensuring that all relevant data is captured and reported.

Integration of Packages

The integration of the Drone, Map, Phase, and Report packages ensures a seamless and effective operation. The Drone package manages command execution, the Map package tracks geographic data, the Phase package orchestrates the scanning process, and the Report package evaluates the success of the mission. Together, they enable the drone to navigate, discover, and report efficiently in challenging environments.

Conclusion

The developed system provides a robust solution for managing drone operations and rescue missions. By combining command transformation, geographic tracking, a structured scanning algorithm, and comprehensive reporting, the system meets the complex requirements of ACME Corp’s Rescue Drone System.
